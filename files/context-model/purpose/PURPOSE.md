# PURPOSE

- This file compresses [VISION.md](../../../VISION.md) and [docs/PURPOSE.md](../../../docs/PURPOSE.md); conflicts resolve toward those files, then get fixed here.
- The mission: collapse the overhead between a human and their artifact — the content is the only thing that matters.
- The thesis: stay close to the artifact; annotate it inline (`::` commands) instead of describing feedback in a chat.
- The wedge shipping today: fmk-indiana, the `::` marker review loop for coding-agent output.
- The destination: the same loop extended to every artifact a human makes with AI — a work management system for the AI-native era.
- The unit of work is a folder: it holds the artifact, the context, and the configuration. We build on where coding agents already operate.
- The unique selling point: knowledge compounds through the loops — this tree IS that mechanism. Feedback given once is never given twice.
- Primary user now: the operator (power-user mode, dogfooded). Later: non-devs ready for agentic workflows, via menulet and buttons.

## Non-goals (decisions, not omissions — reopening one requires a DECISIONS.md entry)

- No own harness: commands hand off to Claude Code / Codex / Cursor. Their tokens, their quota.
- No native apps as an output target: web only — the DOM is inspectable, native is not.
- No solving human-edit version handling on rendered views yet: parked.
- No in-app confirmation dialogs yet: diagnosis loops surface in the agent's chat for now.
- No global cross-project memory yet: this tree is per-repo by design.
