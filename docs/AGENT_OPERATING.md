# AGENT_OPERATING - work loop

## Read first
- [PURPOSE.md](PURPOSE.md) - why this exists.
- [GOAL.md](GOAL.md) - what success looks like.
- [PHASES.md](PHASES.md) - sequencing and roadmap.
- The product spec or architecture doc that owns the area being changed.

## Work loop
- Define success criteria before changing files.
- Prefer existing repo patterns over new abstractions.
- Keep scope tied to the user request.
- State assumptions when uncertain.
- If multiple interpretations exist, present them.
- If a simpler approach exists, say so.
- Push back when direction is heavier than the goal needs.

## End with
- Document learnings, caveats, or new principles in the relevant markdown file.
- Point out consistency issues, violations, caveats, or simpler paths.
- Verify the narrowest useful thing before calling work done.
