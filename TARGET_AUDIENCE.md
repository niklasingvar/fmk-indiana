# TARGET AUDIENCE

## The power agent user


You are a person that use coding agents, for presentaitons, docusments, spreadsheets, scripts, reseradch and coding .... 

You live in a coding agent. You ship features, refactor subsystems, and design architecture — all through an agent loop. You think in prompts, not mouse clicks. You're fast, you're picky, and you hate retyping the same feedback.

You review agent output constantly. Hours of your day are reading markdown, deciding what's right, what's wrong, what to keep, what to fix. You don't want to write essays about it. You want to react — fast — and get back to the loop.


## What you need

| Need | Why |
|------|-----|
| Zero-friction review | Tag a line in one keystroke, not a paragraph of explanation. |
| Agent-ready output | What you mark becomes a prompt the agent understands. No translation step. |
| Speed | Scanning and compiling must be instant. You iterate in seconds, not minutes. |
| Low token overhead | Every byte you send to the agent costs. Compact, compiled bundles only. |
| Filesystem as architecture | You already organize work in folders. The tool must follow that, not fight it. |

## You are

- A driven, tech savvy knowledgeworker who delegates implementation to agents.
- Comfortable in the terminal; happy with a CLI 
- LOVES keyboard shortcuts
- Opinionated about your toolchain. You don't want a platform; you want one sharp tool that does one thing well!

## You are not

- A casual ChatGPT user asking one-off questions.
- Someone who needs a web dashboard to "manage AI workflows."
- A team lead buying enterprise seat licenses.
- Someone who types `Please fix the thing on line 47 where…` every time.

## The promise

Indiana makes the review step of your agent loop near-invisible. You tag. You copy. You paste. The agent acts. The friction is gone — all that's left is the speed of your judgment.
